# Mirror · 产品需求文档 v0.2

> Mirror 是一个 AI 自我觉察与人格成长工具。像一面镜子，帮助用户从每天发生的事里，照见真实的自己。

---

## 1. 产品定位

Mirror 不是心理治疗产品，而是一个**自我觉察工具**。

用户每天写下发生的事、自己的感受和反应，Mirror 会像一面镜子一样，把用户的内在状态反馈回去——不是安慰，不是诊断，而是帮助用户看见自己平时看不见的部分。

**核心假设**：看见，就是改变的开始。

---

## 2. 目标用户

- 对自我探索有兴趣的人
- 经常感到情绪波动、想理解自己为什么会这样的人
- 在关系中反复遇到相似困境的人
- 想成为更稳定、更清醒的自己的人

## 4. 九型人格系统

### 4.1 人格类型（基于 Riso-Hudson 体系）

三种本能中心、九种人格类型：

**腹中心（本能中心）**：
- **1 号 完美型**（The Reformer）：核心动机「我必须做正确的事，才能被接纳」，核心恐惧「做错事、不完美、失控」
- **8 号 领袖型**（The Challenger）：核心动机「我必须强大和掌控，才能避免被伤害」，核心恐惧「脆弱、被控制、被伤害」
- **9 号 和平型**（The Peacemaker）：核心动机「我必须保持和平，才能被接纳」，核心恐惧「冲突、被分离、失去连接」

**心中心（情感中心）**：
- **2 号 助人型**（The Helper）：核心动机「我必须被需要，才能证明我的价值」，核心恐惧「不被需要、被拒绝、孤独」
- **3 号 成就型**（The Achiever）：核心动机「我必须成功，才能被认可」，核心恐惧「失败、不被认可、默默无闻」
- **4 号 独特型**（The Individualist）：核心动机「我必须与众不同，才能找到意义」，核心恐惧「平庸、被抛弃、找不到自我」

**脑中心（思维中心）**：
- **5 号 思考型**（The Investigator）：核心动机「我必须了解一切，才能感到安全」，核心恐惧「无知、被侵犯、资源不足」
- **6 号 忠诚型**（The Loyalist）：核心动机「我必须找到安全和信任，才能安心」，核心恐惧「不安全、被背叛、不确定」
- **7 号 活跃型**（The Enthusiast）：核心动机「我必须保持快乐和自由，才能避免痛苦」，核心恐惧「痛苦、限制、被束缚」

### 4.2 整合与解离方向

每种人格在安全/成长时走向**整合方向**，在压力/焦虑时走向**解离方向**：

| 类型 | 整合方向（健康时） | 解离方向（压力时） |
|---|---|---|
| 1 完美型 | → 7 活跃型 | → 4 独特型 |
| 2 助人型 | → 4 独特型 | → 8 领袖型 |
| 3 成就型 | → 6 忠诚型 | → 9 和平型 |
| 4 独特型 | → 1 完美型 | → 2 助人型 |
| 5 思考型 | → 8 领袖型 | → 7 活跃型 |
| 6 忠诚型 | → 9 和平型 | → 3 成就型 |
| 7 活跃型 | → 5 思考型 | → 1 完美型 |
| 8 领袖型 | → 2 助人型 | → 5 思考型 |
| 9 和平型 | → 3 成就型 | → 6 忠诚型 |

### 4.3 测试设计

- **格式**：强迫选择配对（Forced-Choice Pairs）
- **题量**：72 题（每型约 8 题）
- **原理**：每题呈现两个陈述，用户必须二选一。这种格式比 Likert 量表更能测出无意识偏好。
- **输出**：得分最高的类型为主类型，同时显示第二高分类型（辅助判断翼型）

### 4.4 人格选择器

用户首次打开 Mirror 时，进入人格选择页面：
- 选项 1：开始测试（推荐）
- 选项 2：直接选择自己的类型（9 种可选）

确认人格类型后，后续所有分析都围绕该类型展开。

## 5. 每日记录与分析

### 5.1 输入

- 单一自由书写文本框，不拆分步骤
- 占位提示：「比如：开会时同事打断了我的方案，我没说什么，但心里一直不舒服。回家后又觉得当时自己应该反驳……」
- 支持 `Ctrl/Cmd + Enter` 快速触发
- 「本次不保存」开关（用户可选择某次记录不存入历史）

### 5.2 情绪承接（Echo）

分析开始前，先有一段承接语——接住用户的情绪，不急着分析。

内容结合：
- 用户当前最强烈的情绪
- 用户的人格类型（核心动机 / 核心恐惧）

示例：
> 作为「助人型」，你今天带着「委屈」在走。这种感受对你来说，可能和你的「我必须被需要，才能证明我的价值」有关。我先接住它。

### 5.3 五段式分析框架

每次分析输出五个模块，由浅入深：

**模块 1：情境与事实**
- 区分用户描述中的「客观事实」和「主观解读」
- 事实性关键词：说了、做了、开会、汇报、今天、昨天……
- 解读性关键词：觉得、认为、应该、一定是、是不是、显然……
- 结合人格类型，指出其特有的「解读偏差」
- 目标：帮助用户看到——痛苦往往不在事实本身，而在事实与解读之间的缝隙

**模块 2：人格动力**
- 识别事件如何触动了用户人格的「核心动机」和「核心恐惧」
- 分析用户的「价值感雷达」如何运作
- 匹配用户文本中的人格关键词
- 判断当前状态：走向整合方向 还是 解离方向
- 提示核心盲点

**模块 3：行为模式**
- 分析用户描述中隐含的行为策略
- 检测到的认知模式（10 种）：
  - 讨好倾向
  - 被否定敏感
  - 逃避冲突
  - 过度思虑（思维反刍）
  - 自我攻击
  - 不表达需求
  - 关系焦虑（焦虑型依恋）
  - 拖延 / 回避决定
  - 被「应该」绑架
  - 情绪压抑
- 审视策略的有效性：短期保护了你，还是长期让你失去了什么？

**模块 4：核心需求与渴望**
- 深入表面情绪之下，寻找真正的需求
- 区分「表层渴望」和「深层需求」
- 基于人格类型的「心理陷阱」分析
- 补充事件层面的深层信号：比较、认可、掌控、连接缺失等

**模块 5：建议**
- 1-3 条具体、可操作的行动建议
- 建议来源（多层融合）：
  1. 检测到的认知模式 → 针对性练习
  2. 人格类型 → 专属成长建议
  3. 主导情绪 → 情绪调节建议
- 避免空洞鸡汤，每条建议都有具体步骤
- 结尾一句温暖但有力的鼓励

## 6. 分析引擎：双模式

### 6.1 本地规则引擎（默认，离线可用）

- 基于关键词匹配 + 句子密度加权
- 情绪识别（8 种）、认知模式识别（10 种）、主题领域识别（8 类）
- 人格类型驱动的动态分析生成
- 优点：快、私密、零成本
- 缺点：深度有限、灵活性不足

### 6.2 AI 智能分析（推荐，需配置 API Key）

支持 4 个主流平台：

| 平台 | 可用模型 | 获取地址 |
|---|---|---|
| DeepSeek | V4 Pro / V4 Flash | platform.deepseek.com |
| 硅基流动 SiliconFlow | DeepSeek V3 / Qwen3 235B / GLM-4 9B | cloud.siliconflow.cn |
| 智谱 GLM | GLM-4 Plus / GLM-4 Flash | open.bigmodel.cn |
| 通义千问 Qwen | Qwen Max / Plus / Turbo | dashscope.console.aliyun.com |

**技术细节**：
- 流式输出（SSE），逐字渲染到 5 个分析卡片
- 模块分隔符：`★`
- 系统 Prompt 内置九型人格完整知识 + 五段式分析框架
- 用户 Prompt 包含：用户文本 + 辅助分析信息（情绪/主题/模式/人格类型）
- API 调用失败时自动回退到本地规则引擎
- API Key 仅保存在浏览器 localStorage，不上传任何服务器

## 7. 我的镜像（用户画像）

### 7.1 人格类型展示

- SVG 九型人格图（圆形布局，9 个点位）
- 当前类型高亮
- 整合方向（绿色虚线箭头）和解离方向（红色实线箭头）
- 显示：人格编号、名称、英文名、翼型
- 核心动机、核心恐惧、成长方向、所属中心

### 7.2 我想成为的人

- 5 个预设方向：
  - 情绪稳定的人
  - 自信的人
  - 自律的人
  - 更有行动力的人
  - 更清醒地爱与被爱的人
- 支持自由输入自定义目标
- 设为主目标后，AI 建议会围绕该方向调整

### 7.3 画像摘要

- **统计条**：总记录次数、常出现情绪、主目标状态
- **反复出现的倾向**：可视化强度条（基于历史记录中各认知模式的出现次数）
- **高频主题**：职场/关系、亲密关系、家庭、朋友、自我管理、健康与精力、经济/未来、自我价值

### 7.4 历史记录

- 按时间倒序展示所有记录
- 每条显示日期 + 摘要预览
- 点击可回到「今日」Tab 并加载该记录
- 支持一键清空全部记录

## 8. 成长报告（周报）

- 按自然周自动聚合，无需用户手动生成
- 每周一张周报卡片，包含：
  - 本周记录次数
  - 最常出现的情绪（Top 3）
  - 反复出现的倾向（Top 2）
  - 投入最多心力的主题领域（Top 2）
  - 本周主目标
  - 温和结尾：「你不必每一周都更好。只要每一周都更看清一点自己，就足够了。」

---

## 9. 情绪趋势

### 9.1 情绪分布

- 横向条形图展示各种情绪的总出现次数
- 8 种情绪类别：
  - 愤怒
  - 委屈 / 失落
  - 焦虑 / 不安
  - 羞耻 / 尴尬
  - 平静 / 满足
  - 冲突感 / 矛盾
  - 不被理解
  - 无力感

### 9.2 时间线

- 最近 20 条记录的时间线
- 每条显示日期 + 文本预览 + 情绪标签
- 按时间倒序排列

  - 不太准

## 11. 安全边界

Mirror 不是心理治疗产品。遇到高风险内容时，**不继续做深层分析**，而是给出安全提醒。

### 11.1 识别的风险类型

| 类型 | 说明 |
|---|---|
| 自杀倾向 | 严重绝望、自杀意念、具体计划 |
| 自伤倾向 | 自残、伤害自己的行为 |
| 家庭暴力 / 被控制 | 家暴、被控制、被软禁、精神虐待、PUA |
| 明确人身危险 | 伤害他人、报复社会、同归于尽 |

### 11.2 轻微信号

- 出现 3 个以上「活着好累 / 没有意义 / 没希望 / 绝望 / 撑不下去」等词
- 标记为「强烈无助感」，给出额外关怀

### 11.3 安全提醒内容

- 先共情：「我读到你此刻在很艰难的位置」
- 说明：「你此刻感受到的，不是最终答案」
- 行动建议：先打一个电话给可信任的人
- 资源列表：
  - 全国心理援助热线：400-161-9995（24 小时）
  - 北京心理危机研究与干预中心：010-82951332
  - 紧急情况：拨打 110 / 120

## 12. 品牌与视觉

产品名：Mirror

不设置正式中文名。中文传播中可以使用「照见」「镜子」等表达，但正式品牌保持 Mirror。

### 12.1 双主题系统

Mirror 提供两套配色主题，用户可手动切换，默认跟随系统。

**白天 · 像一本私人笔记**
- 背景：柔白（#FAFAF7）、浅雾灰（#F0EEE9）
- 主色：炭灰（#2C2A28）
- 次文字：灰褐（#5A564F）
- 强调色：淡银（#D8D6D2）、浅紫灰（#B5AFCB）
- 点缀：暖金（#C9A96A，低频次使用）
- 感觉：极简、克制、像一本摊开的私人笔记、光线来自窗外

**夜晚 · 像一场内在探索**
- 背景：黑曜石（#131219）、深灰黑（#1A1924）
- 主色：月白（#E4E1EC）
- 次文字：雾紫灰（#8A8598）
- 强调色：冷紫微光（#7A6FB0 / #B5AFCB）
- 点缀：冷金（#D4BE8E，低频次使用）
- 感觉：神秘、私密、向内、只有自己和镜子

### 12.2 视觉原则

- 低干扰界面：信息密度低，留白充足
- 不拟物化 AI：AI 就是镜子本身，不做人形、头像、卡通
- 镜感通过光影与细微边界表达，而非图形
- 动效克制，以淡入、渐显为主
- 提供一键「白天/夜晚」切换，默认跟随系统

## 13. 隐私原则

Mirror 的隐私承诺：

1. **用户可以随时删除所有数据** —— 一键清空全部记录
2. **所有记录默认私密** —— 没有公开社区、没有分享功能
3. **不做公开社区** —— Mirror 是一面私人的镜子，不是社交平台
4. **AI 分析不用于广告推荐** —— 用户数据仅用于分析本身
5. **用户可以选择某次记录不保存** —— 每次记录都有「本次不保存」开关
6. **API Key 本地存储** —— 如果使用 AI 模式，API Key 仅保存在用户浏览器中

### 技术实现（MVP 阶段）

- 所有数据存储在浏览器 localStorage
- 纯前端应用，无后端服务器
- 不收集任何用户数据

## 17. 品牌语气

Mirror 的语气像一个有洞察力的朋友：

- 温和但不空泛
- 有心理学深度，但不诊断用户
- 可以使用「边界感、依恋模式、防御机制、思维反刍、核心动机」等词，但会解释清楚
- 不说「你就是某某人格」
- 多用「可能有一点……倾向」「这里我看到了……」
- 先稳定情绪，再分析问题

用户完成一次记录后，最重要的感觉是：

> 情绪稳定了一点。我好像更看清自己了一点。
